# Бриф: Золотодозер — нативный прототип на Godot 4

## Назначение документа

ТЗ для реализации через Claude Code. Цель — вертикальный срез: нативная мобильная сцена с физикой монет, собираемая в APK локально и через GitHub Actions. Это НЕ полная игра — это проверка ощущения отзывчивости и производительности нативного стека по сравнению с Three.js-версией.

## Контекст

Существует браузерный прототип (Three.js): бульдозер толкает золотые монеты по платформе (механика coin pusher / gold harvester). Браузерная версия упирается в производительность и input latency. Задача — перенести ядро механики на Godot 4 + Jolt Physics и оценить разницу на реальном устройстве.

## Критерии успеха вертикального среза

1. 200+ динамических монет на сцене при стабильных 60 FPS на среднем Android-устройстве (Snapdragon 7-й серии или аналог).
2. Задержка касания → визуальная реакция ощутимо ниже браузерной версии.
3. Монеты корректно стэкаются (лежат друг на друге без дрожания) и красиво рассыпаются при толчке.
4. APK собирается одной командой локально и автоматически в GitHub Actions при пуше тега.

## 2. Структура проекта

```
zolotodozer/
├── project.godot              # конфиг проекта
├── export_presets.cfg         # Android export preset (без секретов)
├── scenes/
│   ├── main.tscn              # корневая сцена: окружение, камера, свет, UI
│   ├── coin.tscn              # монета: RigidBody3D + коллайдер + меш
│   └── pusher.tscn            # толкатель: AnimatableBody3D
├── scripts/
│   ├── main.gd                # игровой цикл, спавн монет, счёт
│   ├── coin.gd                # логика монеты (звук удара, выход за край)
│   ├── pusher.gd              # возвратно-поступательное движение
│   └── performance_hud.gd     # FPS, число тел, время кадра физики
├── assets/
│   ├── materials/             # золото (metallic PBR), платформа
│   └── audio/                 # звон монет (3-4 вариации)
├── .github/
│   └── workflows/
│       └── build-android.yml
└── README.md                  # как собрать локально, как настроить секреты CI
```

Принцип: максимум логики в `.gd`-скриптах, сцены — минимальные и простые. Это позволяет Claude Code работать с проектом почти полностью без редактора Godot.

## 3. Настройки проекта (project.godot)

- `physics/3d/physics_engine = "Jolt Physics"`
- `physics/common/physics_ticks_per_second = 60` (если на целевом устройстве просядет — снизить до 50, монеты не заметят)
- `rendering/renderer/rendering_method = "mobile"`
- `display/window/handheld/orientation = "portrait"` (как в браузерной версии; уточнить)
- Vsync включён (по умолчанию), `application/run/max_fps = 0` (без лимита, vsync рулит)
- MSAA 2x для мобильного — монеты круглые, лесенки заметны

## 4. Сцена и объекты

### Платформа (StaticBody3D)
- Наклонный лоток с бортами, у нижнего края — обрыв (зона сбора).
- Коллайдеры — только BoxShape3D, никаких trimesh.
- Зона сбора: Area3D под краем платформы, при входе монеты — начисление и `queue_free()` (через пул — см. ниже).

### Монета (coin.tscn)
- `RigidBody3D` → `CollisionShape3D` (CylinderShape3D: радиус 0.12, высота 0.024) → `MeshInstance3D` (CylinderMesh, тот же размер).
- НЕ использовать выпуклую оболочку от меша — только примитив CylinderShape3D.
- Физический материал:
  - mass: 0.05
  - friction: 0.6
  - bounce: 0.1
  - linear_damp: 0.1
  - angular_damp: 0.3 (гасит бесконечное вращение, монеты быстрее «успокаиваются»)
- `can_sleep = true`, `sleeping` агрессивно: покоящиеся стопки не должны есть CPU.
- Continuous collision detection (CCD) ВЫКЛЮЧЕН по умолчанию; включить точечно, только если монеты начнут проскакивать сквозь платформу при падении.
- Звук: при `body_entered` с импульсом выше порога — проиграть случайный из 3-4 сэмплов звона с рандомизацией pitch ±10%. Ограничить: не более 8 одновременных звуков (AudioStreamPlayer3D пул).

### Толкатель (pusher.tscn)
- `AnimatableBody3D` (НЕ StaticBody3D и НЕ RigidBody3D) — движется кодом, физика корректно передаёт импульс монетам.
- Движение синусоидальное в `_physics_process`, период ~2.5 с, амплитуда по размеру платформы.

### Спавн монет
- Пул объектов: предсоздать 250 монет, неактивные хранить выключенными (`freeze = true`, невидимы, вне сцены).
- Тап по экрану → монета активируется над точкой тапа (raycast от камеры до плоскости спавна) с небольшим случайным поворотом и падает.
- Это главная точка оценки отзывчивости: путь «тап → монета появилась» должен быть мгновенным.

### Камера и свет
- Camera3D статичная, угол как в браузерной версии (~50–60° сверху-сзади).
- DirectionalLight3D + Environment с лёгким ambient. Тени: включить, но shadow map 2048 и один каскад — монетам хватит, на мобильном тени дорогие. Если FPS просядет — первым делом отключить тени и сравнить.

### Материал золота
- StandardMaterial3D: metallic 1.0, roughness 0.25, albedo тёплый жёлтый (#E8B923 ориентир).
- Все монеты — ОДИН общий материал (инстансинг batching'а ради).

## 5. Performance HUD (обязательно)

Оверлей в углу экрана:
- FPS (среднее за секунду) и frame time (мс)
- Время физики за кадр (Performance.TIME_PHYSICS_PROCESS)
- Число активных (не спящих) тел
- Кнопка «+50 монет» для стресс-теста

Это инструмент эксперимента: цель прототипа — измерить, а не предположить.

## 6. Локальная сборка

Предварительно (один раз, на десктопе):
1. Установить JDK 17, Android SDK command-line tools (`sdkmanager "platform-tools" "build-tools;34.0.0" "platforms;android-34"`).
2. В Godot: Editor → Manage Export Templates → Download.
3. Editor Settings → Export → Android: указать пути JDK и SDK.
4. Project → Export → Add → Android. Debug keystore генерируется автоматически.

Команда сборки:
```bash
godot --headless --export-debug "Android" build/zolotodozer-debug.apk
```

One-click deploy: телефон по USB с включённой отладкой → кнопка Remote Deploy в редакторе.

## 7. GitHub Actions

Файл `.github/workflows/build-android.yml`. Логика:
- Триггер: push тега `v*` + ручной `workflow_dispatch`.
- Образ: `barichello/godot-ci:4.4.1` (проверить актуальный тег под выбранную версию Godot).
- Шаги: checkout → импорт ресурсов (`godot --headless --import`) → экспорт debug APK → upload artifact.
- Для release-сборки позже: keystore в base64 → GitHub Secrets (`ANDROID_KEYSTORE_BASE64`, `KEYSTORE_PASSWORD`), декодировать в шаге перед экспортом, путь прописать через env-переменные Godot (`GODOT_ANDROID_KEYSTORE_RELEASE_PATH` и т.п.).
- В `export_presets.cfg` секретов быть не должно — он коммитится.

Результат: пуш тега → через ~5-7 минут готовый APK в артефактах релиза, скачиваемый прямо на телефон.

## 8. Порядок реализации (этапы для Claude Code)

1. **Скелет проекта**: project.godot, пустая main.tscn с платформой-боксом, камерой, светом. Запуск на десктопе.
2. **Монета + гравитация**: coin.tscn, тап спавнит монету, она падает и звенит. Проверка стэкинга 50 монет.
3. **Толкатель**: движение, монеты сталкиваются к краю, зона сбора, счётчик.
4. **Пул + HUD**: пул на 250 монет, performance HUD, стресс-тест.
5. **Android export**: локальная сборка APK, тест на устройстве, замеры.
6. **CI**: workflow, сборка по тегу, проверка артефакта.
7. **Тюнинг по результатам замеров**: тени, MSAA, тики физики, damping монет.

Каждый этап завершается работающим состоянием — никаких «соберём всё в конце».
