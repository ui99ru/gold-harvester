# Бэклог

Организован по [ведущему вектору ASMR/залипательность/антистресс](ASMR.md). Эпики — по
приоритету; первый сейчас — **производительность** (без 60 fps вектор недостижим: «60 fps —
обязательное условие потока», ASMR §3.3/§4).

Теги цели: **[godot]** — нативный порт (`godot/`, текущая мобилка), **[web]** — Three.js
(`src/`), **[оба]**.

## Эпик 0 — Производительность 🔴 (блокер вектора) [godot]

**Симптом.** В мобильном приложении (Godot APK) FPS падает до 1–2. HUD на устройстве:
`frame 134 ms`, `physics 58 ms` **при `active bodies 0`**, `coins 1000 / pool 0`,
`≈телефон 464 / 16.7 ms` (бюджет кадра ×28).

**Корень (гипотеза, подтверждается кодом).** Godot-порт **симулирует 1000 реальных
`RigidBody3D`** (`CFG.COIN_N = 1000`, каждая монета — `RigidBody3D` + `CollisionShape3D` +
`MeshInstance3D` + `contact_monitor=true` + `_integrate_forces`, см. `coin.gd`, `coin_pool.gd`).
Это **расходится с ключевым web-инсайтом** [`spec.md §2.4`](spec.md): «массу имитируем, не
симулируем; никакой настоящей физики тысяч тел». `COIN_N=1000` — это потолок web-*позиционной*
модели, а не Jolt. Второй фактор: `physics 58 ms` при 0 активных тел — значит время уходит не в
солвер, а в **GDScript O(N)-поллинг экономики** (`TIME_PHYSICS_PROCESS` включает
`_physics_process`).

Задачи (по возрастанию стоимости — сперва дешёвые замеры, потом архитектура):

- [x] **O1. Разделить замер `physics`: Jolt-движок vs GDScript-экономика.** Живой HUD теперь
  показывает `physics X (jolt Y / gd Z)`: `game.gd` меряет чистое время `_physics_process`
  (`_sim_us_last`), HUD считает `jolt = physics − gd` (`TIME_PHYSICS_PROCESS` включает колбэки
  `_physics_process`). Прочитать числа на устройстве → направить O2/O3. _(Реализовано; сверка —
  на APK.)_
- [ ] **O2. Снять O(N) GDScript-поллинг зон.** `gate.step` (`gates.gd:133` и `:161`),
  `pad.step` (`pads.gd`), `trash.step` (`trash_pad.gd`) каждый физ-тик проходят по **всем 1000**
  `game.pool.get_children()` с чтением `coin.global_position` (вызов в физ-сервер). 2 ворот + пад
  + трэш ≈ 4×1000 итераций/тик в GDScript. Заменить на событийную модель: **`Area3D` с
  `body_entered`/`body_exited`** (движок фильтрует по близости — ручной цикл не нужен), либо
  пространственный индекс / единый проход по списку активных монет. Туда же — freeze-пузырь
  `game.gd:687-697` (проход по 1000 каждые 10 тиков).
- [ ] **O3. Имитация массы вместо полной симуляции (перенос инсайта `spec.md §2.4`).**
  Архитектурный корень. Держать в Jolt только **активные** монеты (рядом с ножом / в движении —
  десятки), осевшие дальние — выводить из физ-мира (`freeze` + сбросить слои коллизий) и
  отрисовывать визуально (см. O4). «Активный пузырь» (`game.gd:682-697`) уже усыпляет дальние —
  расширить до полного вывода из симуляции. Цель: `active bodies` и число живых тел Jolt —
  единицы-десятки, не сотни.
- [ ] **O4. Рендер монет через `MultiMesh`.** Сейчас 1000 отдельных `MeshInstance3D`
  (`coin.gd:82`). Перевести на один `MultiMeshInstance3D` (общие меш/материал уже есть —
  `coin.gd:31-34`). Рассмотреть `MultiMesh` и для кольца скал (80+24 отдельных `SphereMesh`,
  `game.gd:478-508`).
- [ ] **O5. `contact_monitor` только у активных монет.** Сейчас включён всем 1000
  (`coin.gd:75`) — Jolt репортит контакты и зовёт `_integrate_forces` шире, чем нужно.
  Включать по близости/скорости. Механизм точечного переключения уже есть — тумблер «Звон»
  гоняет `contact_monitor` по монетам (`game.gd:251-254`), переиспользовать адресно.
- [ ] **O6. Рендер-бюджет мобайла (A/B на устройстве).** Тумблеры в HUD уже готовы
  (`game.gd:242-258`): тень directional 2048 (`project.godot:42`), `Glow`, `MSAA 2x`. Замерить
  вклад каждого на реальном телефоне (прокси ×8 в HUD покрывает **только физику**, не рендер —
  `performance_hud.gd:9-14`), решить дефолты.
- [ ] **O7. Деградация по железу.** Авто-снижение числа живых монет / отключение glow/тени на
  слабых устройствах (после O3 потолок монет станет управляемым).

**Проверка эпика.** Готовый смоук-харнесс: `godot --headless -- --smoke-stress`
(`game.gd:800-811, 1004-1019`) — осевшая куча 1000, замер `phys`/`gd`/`total` + прокси телефона
×8 + пик тика; `--smoke-bucket` — лаг «монеты в ковше»; `--coins=N` — свип размера пула. Гейтить
каждую задачу этими смоуками **до** пуша. Финальный критерий — **APK на устройстве**: HUD
`≈телефон` ≤ 16.7 мс и стабильные 60 FPS (прокси не покрывает рендер — чекпойнт обязателен).
