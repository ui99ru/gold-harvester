# ЗОЛОТОДОЗЁР · Gold Harvester

Браузерная 3D-игра на Three.js: изометрический бульдозер тяжело сгребает дискретное золото в горку на ноже, толкает его через ворота-множители и сваливает в апгрейд-пады.

**Главная цель проекта — максимально точно воспроизвести физику, геймплей и дизайн из референс-ролика [`demo.mp4`](demo.mp4)** (рекламный креатив Gold and Goblins, AppQuantum). Не сам idle/merge-геймплей Gold and Goblins, а именно ощущение «дозер + золото + ворота + пады» из ролика.

![Референс против текущего рендера](docs/ref_vs_render.png)
<sub>Слева — кадр-эталон из ролика, справа — текущий рендер `index.html`. Сравнение генерится автоматически (`tools/compare.py`).</sub>

## TL;DR

```bash
# 1. собрать корпус эталонов из ролика (один раз)
bash tools/extract.sh

# 2. поставить рендер-харнесс
npm install && npx playwright install chromium

# 3. прогнать цикл: рендер -> метрики -> side-by-side
node tools/shoot.mjs --seed 7 --suffix _after
python tools/compare.py refs/keyframes/kf_hill.png out/shot_hill_after.png --label hill
```

Открыть `index.html` в браузере — играть (тяни мышью/пальцем или WASD).

## Почему так, а не «ещё раз подвигать константы»

Прошлые попытки клонировать ролик тюнились **на глаз** — и не сходились. Корень: ТЗ требует судить картинку по реальному рендеру (§8), но в репозитории не было ни кадров-эталонов, ни рендер-харнесса, ни детерминизма. Каждая правка оценивалась вслепую.

Этот репозиторий построен вокруг **измеримого цикла**:

```
   refs/            tools/shoot.mjs      tools/compare.py        index.html (CFG)
 ┌─────────┐        ┌───────────┐        ┌──────────────┐        ┌───────────┐
 │ эталон  │  ──▶   │  рендер   │  ──▶   │ метрики+глаз │  ──▶   │  правка   │  ──┐
 └─────────┘        └───────────┘        └──────────────┘        └───────────┘    │
      ▲                                                                            │
      └────────────────────────────────────────────────────────────────────────┘
```

Каждая правка цвета/камеры/физики проверяется рендером, сравнивается с эталоном по числам **и** глазами, и принимается только если приближает к рефу. Методика подробно — в [METHOD.md](METHOD.md).

## Целевая фаза ролика

![Контактный лист фаз ролика](docs/phases_contact.png)
<sub>Контактный лист `demo.mp4` (2 кадра/с): первая ячейка — заставка-ковёр (фаза A), дальше геймплей (фаза B).</sub>

Ролик размечен в [refs/phases.md](refs/phases.md) на две фазы:

| Фаза | Время | Что в кадре |
|---|---|---|
| **A. Заставка-ковёр** | 0.0–0.5 c | Бесконечный ковёр золота + самоцветы. «Красивая ложь», **не цель**. |
| **B. Геймплей** | 0.5–28 c | Лавандовый грунт, синий дозер, дискретные крупные монеты, ворота ×N, апгрейд-пады. **Цель клонирования.** |

> `demo.mp4` — захват рекламы: нижние ~25% кадра занимает баннер Google Play и кнопка «Пропустить». При сравнении это автоматически кропается (см. `tools/compare.py`).

**Принцип:** где ролик расходится с текстом ТЗ — **главнее ролик**. Поэтому грунт лавандовый (а не «серый» из ТЗ §2.6), ворота красно-белые (а не синие).

## Игровые системы (текущее состояние)

- **Дозер** — группа боксов, world-aligned камера, курс лерпится к указателю/WASD.
- **Дискретные монеты** — один `InstancedMesh` на 1400 экземпляров, без настоящей физики тысяч тел.
- **Физика «нож-стенка»** (ядро ощущения) — позиционная модель: монеты прижимаются к лицу ножа, выстраиваются по полосам, копятся в стопки, переполнение ссыпается в боковые валы. Дозер встал — куча осталась (тяжесть, не импульс).
- **Стек-грид** — объём горки через грубый 2D-счётчик высот.
- **Ворота-множители** — растят `worth` монеты при проезде, кулдаун на монету.
- **Апгрейд-пады** — поглощают монеты, копят `fill`, на `cost` срабатывает апгрейд (шире нож / захват / ценность).
- **Bloom** — самодостаточный пайплайн на core Three (bright-pass → separable blur → composite), под `try/catch` с откатом на прямой рендер.
- **Звук** — процедурный (clink/chime/engine) на WebAudio.

Подробности и проверенные константы — в [spec.md](spec.md).

## Детерминизм (хуки под харнесс)

`index.html` остаётся single-file, но получил режим воспроизводимого рендера для цикла сравнения:

- `?test=1` — авто-старт без кнопки и без звука.
- `?seed=N` — seeded-RNG (mulberry32) вместо `Math.random` во всех геометрию-формирующих местах. Один seed → идентичный кадр.
- `CFG` — все «магические числа» (палитра, камера, bloom, монеты) собраны вверху файла. Override через `?key=val` или `window.__cfg` — можно свипать параметр без правки файла.
- `window.__sim` — API для харнесса: `step(dt)`, `run(n,dt)`, `setTarget`, `setPose`, `state`, `render()`.

Обычный геймплей (без `?test`) этим не затронут.

## Прогресс клонирования

Первый прогон цикла — **палитра** (грунт + золото + экспозиция). SCORE — суммарная дельта по эталону, меньше = ближе:

![Рендер до и после первого прогона](docs/before_after.png)
<sub>Один и тот же бет `hill`: до — серый грунт и бледное золото; после — лавандовый грунт и насыщенное золото, подогнанные под реф.</sub>

| бет | before | after | улучшение |
|---|---:|---:|---:|
| establish | 287.2 | 99.1 | −65% |
| hill | 272.8 | 90.8 | −67% |
| spread | 278.5 | 148.5 | −47% |
| pad | 298.4 | 160.4 | −46% |

Запечено в `CFG`: `exposure .82→.56`, `groundColor →0x504c5e` (лавандовый), `coinColor →0xdc7407`, насыщенность/металл/шероховатость золота под реф.

### Дальше (по убыванию разрыва)

1. **Камера/композиция** — реф площе и шире, дозер мельче в кадре.
2. **Ворота** — крупный красный шеврон ×N вместо сине-голубой завесы.
3. **Сценарий бет** в `shoot.mjs` под тайминг ролика.
4. **Пады-рампы** с надписью «UPGRADE» и силуэтом награды.
5. **Juice** — наклон/зум камеры по скорости, искры, пыль, хаптика.

## Стек и требования

- **Игра:** Three.js r128 (CDN), чистый single-file `index.html`. Целевой перф — 60 fps на среднем телефоне.
- **Тулинг:** Node 18+ (Playwright), Python 3.9+ (Pillow, numpy), ffmpeg.

## Лицензия

Учебный прототип/клон для воспроизведения ощущения из референса. `demo.mp4` принадлежит правообладателю Gold and Goblins (AppQuantum) и используется как референс.
